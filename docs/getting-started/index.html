Getting Started

Set up the app, run native builds, and validate that the enhanced WatermelonDB build is active.

Home Getting Started Reactive API Architecture Troubleshooting

1. Install dependencies

npm install

This app installs WatermelonDB from a vendored tarball so clones can run without depending on local sibling paths. 

vendor/nozbe-watermelondb-0.28.1-0.tgz

2. Choose DB source

Default (vendored tarball):

npm run setup:db:vendor

Use published npm package when available:

npm run setup:db:npm
# or pin exact version
./scripts/setup-enhanced-db.sh npm @onchezz/hyperdb@0.28.1-0.enhanced.0

3. Run native app

# Android
npx expo run:android

# iOS
npx expo run:ios

4. Compile checks

npx tsc --noEmit
CI=1 npx expo export --clear --platform android

5. Verify reactive flow

  • Create a task from the input in the app.
  • Observe the task list update immediately without writing manual subscription logic.
  • Tap a task to toggle is_done and confirm UI updates from local DB state.
  • Use Seed Demo to insert sample rows once only when table is empty.

6. Build and publish package from fork

# in hyperDb repo
npm run build:enhanced:npm
npm run publish:enhanced:npm

Detailed packaging guide: NPM_PACKAGE_SETUP.md

7. Update vendored tarball from fork

# from hyperDb clone
npm install
npm run build
cd dist
npm pack

# copy new tgz into this repo vendor/ then
npm install
