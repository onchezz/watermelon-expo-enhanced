Getting Started

Set up the app, run native builds, and validate that the enhanced WatermelonDB build is active.

Home Getting Started Reactive API Architecture Troubleshooting

1. Install dependencies

npm install

This app installs WatermelonDB from a vendored tarball so clones can run without depending on local sibling paths. 

vendor/nozbe-watermelondb-0.28.1-0.tgz

2. Run native app

# Android
npx expo run:android

# iOS
npx expo run:ios

3. Compile checks

npx tsc --noEmit
CI=1 npx expo export --clear --platform android

4. Verify reactive flow

  • Create a task from the input in the app.
  • Observe the task list update immediately without writing manual subscription logic.
  • Tap a task to toggle is_done and confirm UI updates from local DB state.

5. Update to a newer enhanced DB package

# from watermelondb-enhanced clone
npm install
npm run build
cd dist
npm pack

# copy new tgz into this repo vendor/ then
npm install
