Reactive API Usage

Use WatermelonDB in a Supabase-like style while keeping local-first behavior and performance.

Home Getting Started Reactive API Architecture Troubleshooting

Create the client

import { createReactiveClient } from '@nozbe/watermelondb'
import { database } from './index'

export const reactive = createReactiveClient(database)

Supabase-style local-first service

import { supabaseLocal } from './src/database/localFirst'

await supabaseLocal.tasks.create('Draft release notes')
await supabaseLocal.tasks.toggle(taskId, false)
await supabaseLocal.tasks.rename(taskId, 'Publish docs')
await supabaseLocal.tasks.clearCompleted()

Read reactively in a component

import { useReactiveQuery } from '@nozbe/watermelondb/reactive/react'
import { supabaseLocal } from './src/database/localFirst'

const { data, isLoading, error } = useReactiveQuery(
  () => supabaseLocal.tasks.query({ status: 'open', limit: 100 }),
  ['open'],
)

No manual useEffect subscription boilerplate is required.

Filter + paginate query

const page = 0
const pageSize = 20

const tasks = await supabaseLocal.tasks.fetch({
  status: 'open',
  from: page * pageSize,
  to: page * pageSize + pageSize - 1,
})

Subscribe manually (without hooks)

const unsubscribe = supabaseLocal.tasks.subscribe(
  { status: 'all' },
  ({ data, error }) => {
    if (error) return
    console.log('rows', data?.length ?? 0)
  },
)

// later
unsubscribe()

Seed demo data once

await supabaseLocal.tasks.seedDemoIfEmpty()

Important local-first note

WatermelonDB is local. Query fields like project_id are only valid if that column exists in your local schema. There is no remote table unless your app adds sync logic.
