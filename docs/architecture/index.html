Flow overview

UI (App.tsx)
  -> useReactiveQuery(() => supabaseLocal.tasks.query(...))
Local-first Service (src/database/localFirst.ts)
  -> create / rename / toggle / clearCompleted / seedDemoIfEmpty
Reactive Client (src/database/reactive.ts)
  -> createReactiveClient(database)
Database (src/database/index.ts)
  -> new Database({ adapter, modelClasses })
SQLite Adapter
  -> Watermelon local SQLite storage

Core files

File Responsibility
src/database/schema.ts Defines app schema and table columns.
src/database/models/Task.ts Task model read/write helpers around raw record fields.
src/database/index.ts Initializes SQLite adapter and Watermelon database instance.
src/database/reactive.ts Exports reactive query client and row types.
src/database/localFirst.ts Supabase-style API wrapper around WatermelonDB for working local-first CRUD and query helpers.
src/database/examples/supabaseLocalExamples.ts Executable examples showing create/list/toggle/rename/subscribe usage.
App.tsx Uses reactive query + mutation calls and renders task list.

Design choices

  • Use package entrypoints only (no internal src/ imports).
  • Keep writes in a compact command style similar to Supabase.
  • Avoid manual subscription lifecycles in component code.
  • Ship vendored enhanced package tarball for portable installs.
